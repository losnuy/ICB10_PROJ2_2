# **데이터 기반 채용 미스매치 해소를 위한 스트림릿 대시보드 통합 기획 및 안티그래비티 마스터 프롬프트 설계**

## **서론: 정보 비대칭 해소와 바이브 코딩 패러다임의 결합**

오늘날의 채용 시장은 구직자의 스펙 상향 평준화와 기업의 심각한 구인난이 공존하는 모순적인 상황에 직면해 있다. 이러한 현상의 근본적인 원인은 철저히 '정보 비대칭(Information Asymmetry)'에 기인한다. 구직자는 기업이 실제 실무에서 크게 요구하지 않는 범용 자격증(예: 컴퓨터활용능력 1급, 일반적인 어학 고득점)에 과도한 시간과 비용을 투자하며 취업 준비 기간을 장기화하고 있다1. 반면, 기업의 인사팀은 직무의 핵심 역량과 무관한 '허수 지원자'들을 스크리닝하는 데 막대한 리소스를 소모하며, 진성 인재를 놓치는 기회비용을 지불하고 있다1.
이러한 노동 시장의 비효율을 해결하기 위해 데이터 기반의 HR 애널리틱스 대시보드 도입이 기업과 교육 기관 전반에서 가속화되고 있다3. 특히, 최근 데이터 분석 비전공자들이 자연어 프롬프트를 통해 인공지능 에이전트와 협업하여 소프트웨어를 구축하는 '바이브 코딩(Vibe Coding)' 방법론이 부상하면서, 복잡한 시스템 기획과 구현의 장벽이 크게 낮아졌다6. 안티그래비티(Antigravity)와 같은 자율형 AI 코딩 에이전트를 활용하면 코드 단위의 지식 없이도 고도화된 웹 애플리케이션을 배포할 수 있으나, 분산된 형태의 대시보드들을 하나로 통합하고 알고리즘을 고도화하기 위해서는 매우 정교한 아키텍처 설계와 프롬프트 엔지니어링이 선행되어야 한다7.
본 보고서는 수집된 1,000건 이상의 사람인(Saramin) 직무별 채용 공고 데이터를 기반으로, 구직자와 구인자 양측의 니즈를 동시에 충족하는 '스펙-직무 매칭 및 가이드형 솔루션'의 상세 기획안을 제시한다. 기존에 각 팀원이 개별적으로 구축한 세 개의 스트림릿(Streamlit) 대시보드 구조를 단일 다중 페이지 애플리케이션(Multipage App)으로 병합하는 기술적 방법론을 논의하고, 텍스트 마이닝과 비지도 학습을 결합한 새로운 핵심 기능을 제안한다. 결론적으로, 데이터 기획 경험이 부족한 프로젝트 팀이 안티그래비티 환경에 직접 입력하여 오류 없이 상용화 수준의 대시보드를 구축할 수 있는 단계별 '마스터 프롬프트 아키텍처'를 도출한다.

## **데이터 아키텍처 및 백엔드 파이프라인 최적화**

분산된 대시보드를 하나로 합치고 실시간 인터랙션을 지원하기 위해서는 가장 먼저 데이터의 저장 및 호출 구조를 중앙집중화해야 한다. 정적인 CSV 파일들에 의존하던 기존 방식을 탈피하여, SQLite3를 마스터 테이블로 활용하는 아키텍처 전환이 필요하다.

### **스레드(Thread) 안정성 및 SQLite3 동시성 제어**

스트림릿은 웹 브라우저에서 사용자의 클릭이나 입력과 같은 인터랙션이 발생할 때마다 파이썬 스크립트를 위에서부터 아래로 완전히 재실행하는 독특한 렌더링 주기를 가진다9. 이 과정에서 가장 빈번하게 발생하는 데이터베이스 계층의 치명적인 오류는 sqlite3.ProgrammingError: SQLite objects created in a thread can only be used in that same thread이다11. 스트림릿의 실행 모델은 매 재실행 시 새로운 스레드를 할당할 가능성이 높은 반면, SQLite3 모듈은 기본적으로 연결(Connection) 객체를 생성한 스레드 외의 다른 스레드에서 해당 객체에 접근하는 것을 엄격히 차단하기 때문이다11.
이러한 스레드 충돌을 방지하기 위해, 안티그래비티 에이전트가 데이터베이스 연결 객체를 생성할 때 반드시 check\_same\_thread=False 인자를 부여하도록 프롬프트 레벨에서 강제해야 한다11. 이를 통해 반환된 연결 객체는 여러 스레드에 걸쳐 공유될 수 있다. 단, 다중 사용자 환경에서 동시 쓰기(Write) 작업이 발생할 경우 데이터 무결성이 훼손될 수 있으므로, 본 대시보드와 같이 '읽기(Read) 전용' 분석을 수행하는 아키텍처에서 가장 적합하고 안전한 접근법이다.

### **데이터 정제 및 동의어 처리(Thesaurus) 파이프라인**

채용 공고의 '자격요건'과 '우대사항'은 기업마다 작성하는 양식이 다르고 비정형 텍스트로 구성되어 있어, 단순한 문자열 매칭으로는 알고리즘의 정확도를 보장할 수 없다1. 따라서 백엔드 파이프라인 단계에서 텍스트 정제(Text Cleansing)와 동의어 사전(Thesaurus) 매핑 프로세스가 반드시 수반되어야 한다1.
예를 들어, '정처기', '정보처리기사', '정보 처리 기사'는 모두 동일한 자격증을 지칭하며, '스마트A', 'Smart A', '더존' 역시 동일한 회계 실무 툴을 의미한다. 정규표현식과 자연어 처리(NLP) 라이브러리를 활용하여 이러한 비정형 명사 구문을 추출하고 사전 정의된 마스터 키워드로 치환하는 과정이 1차적으로 진행된다1. 이 정제 작업이 완료된 후, 자격증, 어학, 실무 툴, 세부 직무 경험 등의 항목들이 바이너리(0 또는 1\) 형태의 매트릭스로 변환되어 SQLite3 마스터 테이블에 적재되어야 한다.

## **머신러닝 및 자연어 처리(NLP) 기반 코어 알고리즘 설계**

통합 대시보드의 비즈니스 로직은 사용자와 채용 공고 간의 거리를 측정하는 수학적 모델에 기반한다. 팀이 기존에 구상한 단일 매칭 방식을 넘어, 보다 정교한 통계적 검증과 군집화 기법을 도입하여 대시보드의 효용성을 극대화한다.

### **TF-IDF 기반 스펙 가중치 이원화 모델**

구직자의 스펙과 채용 공고를 매칭할 때, 단순 빈도수(Term Frequency)만을 고려하면 치명적인 왜곡이 발생한다1. '엑셀 활용 가능자', '컴퓨터활용능력 우수자'와 같은 단어는 대부분의 직무 공고에 관습적으로 포함된다. 만약 이러한 단어에 높은 점수를 부여한다면, 컴활 자격증만 보유한 구직자가 재무회계 직무에 90점의 적합도를 받는 오류가 발생한다1.
이 문제를 해결하기 위해 문서 빈도 역역산(TF-IDF, Term Frequency-Inverse Document Frequency) 기법을 핵심 알고리즘으로 채택한다1. 알고리즘은 전체 1,000건의 채용 공고 풀(Pool)에서 특정 단어가 얼마나 흔하게 등장하는지를 계산하여 가중치를 조정한다. 모든 공고에 등장하는 범용 스펙(Base Spec)은 IDF 값이 0에 수렴하여 가중치가 대폭 삭감된다. 반면, 특정 고연봉 공고나 전문 직무 공고에만 국지적으로 등장하는 핵심 스펙(Core Spec, 예: IFRS, SAP, 파이썬 데이터 분석)은 매우 높은 가중치를 획득하게 된다1. 이 모델을 통해 대시보드는 "단순 범용 자격증에 목매지 말고, 해당 직무의 Core 스펙을 취득하라"는 데이터 기반의 직설적인 가이드를 제공할 수 있다.

### **코사인 유사도(Cosine Similarity)를 활용한 적합도 예측**

구직자가 대시보드에 자신의 스펙을 입력하면, 시스템은 이를 하나의 다차원 벡터로 변환한다. 이 유저 벡터와 SQLite3 데이터베이스 내에 저장된 1,000개의 채용 공고 벡터(TF-IDF 가중치가 반영된 벡터) 간의 방향적 유사성을 코사인 유사도로 산출한다16. 코사인 유사도는 두 벡터 사이의 각도를 기반으로 0에서 1 사이의 스코어를 반환하며, 이는 "내 직무 적합도: 85점"과 같은 직관적인 100점 만점 척도로 환산되어 사용자에게 실시간으로 제공된다1. 이 연산은 행렬 내적을 통해 이루어지므로, 스트림릿 환경에서도 지연 없이 즉각적인 결과를 출력할 수 있다.

### **K-Means 기반 기업 채용 유형 자동 군집화 (Clustering)**

기존 프로젝트 구상에서는 인사, 재무, 회계 등 표면적인 직무 이름이나 대기업, 중소기업 등 하드코딩된 기준으로 공고를 분류하려 했다. 그러나 실제 데이터의 양상은 훨씬 복잡하다1. 동일한 '회계' 직무라 하더라도, 어떤 중소기업은 인사와 총무, 급여 계산을 모두 아우르는 '경영지원 멀티태스킹' 역량을 요구하는 반면, 다른 기업은 오직 '외감 수감'과 '연결재무제표'만을 요구한다1.
이에 따라 대시보드 통합 시 K-Means 군집화(Clustering)와 같은 비지도 학습 알고리즘을 백엔드 파이프라인에 추가할 것을 제안한다1. 알고리즘이 1,000건의 공고에 담긴 자격요건 텍스트 패턴을 스스로 분석하여, 데이터 내재적 특성에 따라 3\~4개의 '진짜 기업 유형'으로 클러스터링한다1. 데이터가 스스로 정의한 이 군집 정보를 바탕으로, 구직자는 자신이 지향하는 기업 스타일에 따라 전혀 다른 '스펙 피벗(Pivot)' 전략을 세울 수 있다1.

## **참신하고 실용적인 신규 분석 기능 제안**

각 팀원이 구현한 기존 대시보드의 단순 통계 뷰를 뛰어넘어, 취업 시장의 정보 비대칭을 꼬집는 고도화된 인사이트 기능들을 새로운 탭에 통합한다. 이는 구직자와 인사담당자 모두에게 강력한 실용성을 제공한다.

### **연봉 상관관계 기반 스펙 부스터(ROI) 시뮬레이션**

수집된 사람인 데이터 내의 '최소 연봉(sal\_min)' 데이터를 활용하여, 특정 자격증이나 스펙이 실제 몸값 상승에 통계적으로 기여하는지 분석하는 기능을 추가한다1. 카이제곱 검정(![][image1] Test) 및 T-test를 활용하여, 특정 자격증(예: 재경관리사)을 요구하는 공고 그룹과 그렇지 않은 공고 그룹 간의 평균 연봉 차이를 박스 플롯(Box Plot)으로 시각화한다1. 구직자에게는 "이 자격증을 취득할 경우 지원 가능한 평균 연봉이 OOO만 원 상승합니다"라는 가성비(ROI) 지표를 제공하여, 막연한 스펙 쌓기를 방지한다.

### **인사팀용 '허수 자석(Spam Magnet)' 진단 및 가이드**

인사담당자를 위한 탭에는 채용 공고의 '진정성'을 평가하는 기능을 도입한다. 인사팀이 작성 중인 채용 공고(JD) 텍스트를 대시보드에 붙여넣으면, 알고리즘이 공고 내 범용 단어(Base Spec)와 핵심 단어(Core Spec)의 비율을 실시간으로 분석한다1.
만약 공고가 모호하고 범용적인 단어로만 채워져 있다면, "이 공고는 허수 지원자 유입 위험도 85%의 '허수 자석' 공고입니다"라는 경고를 동적으로 출력한다1. 나아가 앞서 수행한 TF-IDF 분석 결과를 바탕으로, 해당 산업군 내 고연봉 우량 기업들이 공통적으로 우대하는 '핵심 실무 키워드 TOP 3'를 추천하여 JD 수정을 유도한다1. 이는 인사팀의 이력서 스크리닝 리소스를 획기적으로 감축시키는 실용적 가치를 지닌다.

## **스트림릿 UI/UX 및 전역 상태 관리 아키텍처**

초급자들이 개발한 세 개의 서로 다른 대시보드를 하나로 매끄럽게 연결하고 상용 애플리케이션 수준의 사용자 경험(UX)을 제공하기 위해, 스트림릿의 최신 기능과 최적화 기법을 전면적으로 적용해야 한다.

### **멀티페이지 라우팅 및 세션 상태(Session State) 동기화**

독립된 스크립트들을 하나로 묶기 위해 st.navigation과 st.Page 함수를 활용한 동적 라우팅 방식을 채택한다23. 이 방식은 탭(st.tabs)으로 모든 기능을 묶어 한 번에 렌더링하는 것보다 초기 로딩 속도를 크게 개선하며, 모듈화된 코드 관리를 가능하게 한다26.
멀티페이지 앱 환경에서 가장 중요한 것은 전역 상태 관리다. 구직자가 첫 번째 페이지에서 입력한 스펙 데이터나 인사담당자가 설정한 필터 조건은 페이지를 전환할 때마다 증발할 위험이 있다. 이를 방지하기 위해 st.session\_state 객체를 초기화하는 로직이 메인 app.py 최상단에 위치해야 한다29.

Python
if "user\_specs" not in st.session\_state:
```
st.session
```

\_state.user\_specs \= {"certs": \[\], "tools": \[\], "experience": 0}

프롬프트를 통해 에이전트가 위와 같은 세션 상태 초기화 패턴을 필수적으로 적용하도록 지시함으로써, 데이터 유실 없이 유기적으로 연결된 다중 페이지 워크플로우를 보장한다9.

### **캐싱(Caching) 전략을 통한 성능 극대화**

SQLite3 마스터 테이블에서 1,000건 이상의 데이터를 불러오고 코사인 유사도 행렬을 연산하는 작업은 컴퓨팅 리소스를 많이 소모한다. 스트림릿의 특성상 사용자가 버튼 하나만 클릭해도 전체 스크립트가 재실행되므로, 적절한 캐싱 없이는 대시보드의 사용성이 심각하게 저하된다9.
데이터베이스 연결 객체와 같이 전역적으로 공유되며 변경되지 않아야 하는 객체에는 @st.cache\_resource를 적용하고, 쿼리된 데이터프레임이나 모델 추론 결과 등 직렬화 가능한 데이터에는 @st.cache\_data를 적용하여 메모리 효율을 극대화해야 한다9. 이를 통해 마치 실시간으로 통신하는 것과 같은 쾌적한 렌더링 속도를 확보할 수 있다.


| 캐싱 데코레이터     | 적용 대상 객체                   | 동작 원리 및 특징                                                                          |
| :-------------------- | :--------------------------------- | :------------------------------------------------------------------------------------------- |
| @st.cache\_resource | SQLite3 Database Connection      | 역직렬화할 수 없는 전역 리소스 보존. 모든 사용자/세션이 동일 객체 공유32                   |
| @st.cache\_data     | Query Results, Pandas DataFrames | 반환값을 Pickle로 직렬화하여 저장. 연산된 데이터를 메모리 복사본으로 제공하여 안전성 확보9 |

### **Plotly 및 Metric Card를 활용한 시각화 디자인**

데이터의 의미를 명확히 전달하기 위해 시각화 라이브러리인 Plotly를 전면 도입한다. 직무 역량의 갭(Gap)을 시각화하는 데에는 px.line\_polar를 활용한 레이더 차트(Radar Chart)가 가장 적합하다21. 다중 트레이스를 겹쳐 구직자 현재 수준과 기업 요구 수준을 직관적으로 대조한다. 아울러 기업 수요와 구직자 공급의 불균형을 보여주는 바 차트(Bar Chart) 구현 시, 여백으로 인한 시각적 분산 현상을 막기 위해 bargap=0 또는 bargap=0.1 옵션을 레이아웃에 명시적으로 주입한다36.
핵심 성과 지표(KPI)는 st.metric 위젯을 사용하여 대시보드 최상단에 굵게 강조한다39. '매칭 적합도 예측 점수'나 '예상 연봉 상승액' 등을 표기할 때, delta 옵션을 활용하여 이전 스펙 대비 상승/하락 폭을 녹색(Green) 화살표 애니메이션과 함께 표시함으로써 게이미피케이션(Gamification) 효과를 유도한다3. 또한, HTML/CSS 인젝션 기법이나 st\_yled와 같은 서드파티 래퍼 패키지 구조를 프롬프트에 포함하여, 투박한 기본 컨테이너 대신 테두리와 그림자가 적용된 세련된 '카드(Card)' 형태의 UI 레이아웃을 구성한다41.

## **안티그래비티(Antigravity) 다중 페이지 병합 마스터 프롬프트**

비전공자로 구성된 프로젝트 팀이 복잡한 텍스트 마이닝부터 동적 다중 페이지 UI까지 한 번에 오류 없이 구현할 수 있도록, 안티그래비티 에이전트에 직접 복사하여 붙여넣을 수 있는 단계별 '마스터 프롬프트'를 설계하였다. 이 프롬프트는 에이전트가 문맥을 잃지 않도록 데이터 정제(Phase 1), 아키텍처 및 상태 관리(Phase 2), 시각화 및 로직(Phase 3)으로 분리하여 순차적으로 지시하도록 구성되었다.

### **\[Phase 1\] 백엔드 데이터베이스 최적화 및 NLP 알고리즘 구축 프롬프트**

이 프롬프트는 로우 데이터를 정제하고, TF-IDF와 코사인 유사도를 위한 벡터를 계산하여 스레드 안전성을 갖춘 SQLite3 데이터베이스를 구축하는 데 목적이 있다.**\[시스템 역할 및 컨텍스트\]**너는 최고 수준의 HR 데이터 사이언티스트이자 스트림릿 아키텍트야. 내 작업 폴더에 있는 saramin\_crawled\_data.csv 파일을 분석하고 정제하여, 다중 페이지 스트림릿 대시보드에 활용될 마스터 SQLite3 데이터베이스를 구축하는 파이썬 스크립트(db\_pipeline.py)를 작성하고 실행해줘.**\[데이터 정제 및 NLP 처리 요구사항\]**

1. 채용 공고의 '자격요건' 및 '우대사항' 컬럼에 대해 텍스트 전처리를 수행해. '정처기'는 '정보처리기사'로, '스마트A'는 '더존'으로 통일하는 등 파편화된 동의어를 하나로 묶는 시소러스(Thesaurus) 딕셔너리를 코드로 정의하고 적용해줘.
2. 정제된 텍스트를 바탕으로 TF-IDF (Term Frequency-Inverse Document Frequency) 벡터라이저를 훈련시켜서, 각 채용 공고의 텍스트를 고차원 스펙 벡터로 변환해줘. 모든 공고에 등장하는 단어는 가중치를 낮추고, 특정 공고에만 나오는 단어는 가중치를 높이는 방식을 엄격하게 적용해.
3. K-Means 군집화 알고리즘을 사용해 공고들을 특성(멀티태스킹형, 전문심화형 등)에 따라 3개의 클러스터로 자동 분류하고 company\_cluster 컬럼에 라벨링해줘.

**\[SQLite3 적재 요구사항\]**

1. 가공 완료된 데이터프레임을 hr\_master.db 파일의 job\_postings 테이블로 저장해.
2. 추후 스트림릿의 다중 스레드 환경에서 데이터베이스 잠금(Lock) 에러가 발생하지 않도록, sqlite3.connect('hr\_master.db', check\_same\_thread=False) 옵션을 반드시 명시하여 연결 객체를 생성해줘.

### **\[Phase 2\] 스트림릿 멀티페이지 라우팅 및 세션 상태 구조화 프롬프트**

이 프롬프트는 3개의 흩어진 대시보드를 하나로 묶는 app.py 뼈대를 작성하고, 사용자가 입력한 데이터가 증발하지 않도록 전역 세션 상태를 설정한다.**\[UI/UX 아키텍처 및 라우팅 요구사항\]**Phase 1에서 만든 hr\_master.db를 읽어오는 통합 스트림릿 앱(app.py)의 뼈대를 작성해줘.

1. st.navigation과 st.Page 함수를 사용하여 '구직자 스펙 진단 페이지'와 '인사팀 허수 필터링 페이지'를 각각 독립된 모듈로 분리하여 내비게이션 사이드바를 구성해줘.
2. 스크립트 최상단에 st.session\_state를 초기화하는 블록을 작성해. 사용자의 보유 자격증 리스트, 어학 점수, 타겟 직무 등 입력값이 페이지를 이동하거나 위젯과 상호작용하더라도 절대 초기화되지 않도록 해줘.

**\[캐싱 및 데이터 로딩 요구사항\]**

1. 데이터베이스 연결 함수를 정의하고, 반드시 @st.cache\_resource 데코레이터를 적용하여 커넥션 풀을 관리해.
2. 쿼리를 통해 데이터를 불러오고 판다스 데이터프레임으로 반환하는 함수에는 @st.cache\_data(ttl=3600)를 적용하여, 유저 인터랙션마다 쿼리가 재실행되는 성능 저하 현상을 원천적으로 차단해줘.

### **\[Phase 3\] 코어 로직 매칭 및 Plotly 고급 시각화 프롬프트**

마지막 프롬프트는 데이터와 알고리즘을 화면에 시각적으로 렌더링하고, 정보 비대칭을 꼬집는 실용적 인사이트를 출력한다.**\[구직자 페이지 기능 요구사항\]**

1. **적합도 연산:** 사용자가 st.session\_state에 입력한 보유 스펙을 벡터화한 뒤, 데이터베이스 내 공고 벡터들과의 코사인 유사도(Cosine Similarity)를 계산해. 이 점수를 st.metric을 사용해 "내 직무 합격 적합도: 85점"과 같이 크게 표시하고, delta 옵션을 써서 이전 대비 상승폭을 초록색 화살표로 보여줘.
2. **레이더 차트:** 사용자가 입력한 스펙 수준과, 타겟하는 company\_cluster의 평균 요구 스펙 수준을 오버레이하여 비교하는 Plotly 레이더 차트(px.line\_polar)를 그려줘.
3. **스펙 피벗 가이드:** 유저가 보유하지 않은 스펙 중 TF-IDF 가중치가 가장 높은 핵심 스펙 TOP 3를 추출하여, "이 스펙 취득 시 적합도 예상 상승폭: \+15%"라는 시뮬레이션 가이드를 텍스트로 제시해.

**\[인사팀 페이지 기능 요구사항\]**

1. **스펙 갭(Gap) 분석:** 시장 전체에서 지원자들이 이력서에 채워 넣는 낭비성 스펙 빈도와, 해당 직무가 실제로 요구하는 스펙 빈도를 대조하는 Plotly 양방향 바 차트(px.bar)를 렌더링해. 바 사이의 여백이 없도록 fig.update\_layout(bargap=0.1) 설정을 명시해줘.
2. **허수 자석 예측기:** 인사팀 사용자가 채용 공고(JD) 텍스트를 붙여넣으면, 알고리즘이 분석하여 "허수 지원자 유입 위험도: OOO%"를 메트릭으로 출력해.
3. **디자인 마감:** 화면 내의 각 기능 컨테이너들이 투박하지 않게 보이도록, 마크다운의 \<style\> 태그 인젝션을 사용해 컨테이너 모서리를 둥글게 하고 옅은 그림자를 넣는 CSS 코드를 추가해줘. 완성된 app.py를 안티그래비티 로컬 환경에서 실행하여 정상 구동을 검증해.

## **결론 및 발전 방향**

본 통합 기획안은 비전공자 프로젝트 팀이 직면한 파편화된 코드와 단순 통계 위주의 기능 한계를 극복하고, 구직자와 구인자 간의 정보 비대칭을 혁신적으로 해소할 수 있는 로드맵을 제공한다. 기존의 단순 빈도수 집계를 뛰어넘어, TF-IDF와 코사인 유사도를 접목한 다차원 텍스트 매칭과 K-Means를 통한 기업 유형 군집화를 도입함으로써 알고리즘의 신뢰성과 깊이를 대폭 강화하였다1. 또한, SQLite3 스레드 충돌 제어, 세션 상태를 활용한 다중 페이지 라우팅, 고효율 캐싱 전략 등 스트림릿 생태계의 기술적 병목을 해결하는 아키텍처를 세밀하게 설계하였다9.
데이터 분석 초급자들은 위에서 제시된 세 단계의 마스터 프롬프트를 안티그래비티(Antigravity) 에이전트에 순차적으로 입력함으로써, 복잡한 코딩 문법에 매몰되지 않고 기획 의도에 완벽하게 부합하는 인터랙티브 대시보드를 성공적으로 배포할 수 있을 것이다6. 향후 본 애플리케이션은 네이버 데이터랩 API 등 외부 데이터 소스와의 실시간 연결을 통한 시계열 동적 업데이트 기능이나, 오픈소스 경량 대형언어모델(LLM)을 연동한 자기소개서 맥락 피드백 기능 등으로 그 영역을 더욱 확장해 나갈 수 있다2. 본 보고서가 바이브 코딩 실무 코스의 프로젝트 병합 성공은 물론, 실효성 있는 HR 애널리틱스 프로덕트로 도약하는 강력한 지침서가 되기를 기대한다.
