# 서울시 행정동별 생활인구 데이터 변환 및 최적화 보고서

본 보고서는 `@seoul-pops/data/LOCAL_PEOPLE_DONG_202606.zip` 원본 데이터를 로드하여 분석에 적합한 Tidy Data 형태로 변환하고, 메모리 및 파일 크기를 최적화(Downcasting)하여 `.parquet` 파일로 저장한 결과와 이에 따른 성능 변화를 분석합니다.

---

## 1. 데이터 처리 개요

### 1.1 Tidy Data(깔끔한 데이터) 변환
- **구조적 개선**: 원본 데이터는 성별과 연령대가 컬럼 이름에 복합적으로 포함된 **Wide-format(넓은 형태)** 구조입니다. 이는 데이터 분석 시 필터링이나 그룹화 작업을 어렵게 만듭니다.
- **해결책**: `pandas.melt()` 메서드를 활용하여 기준일, 시간대, 행정동을 고정(ID 변수)하고 28개의 성별/연령대 컬럼을 단일 행으로 녹여내는 **Long-format(긴 형태)** 구조로 변환하였습니다.
- **컬럼 분리**: 녹여낸 문자열 컬럼(예: `'남자0세부터9세생활인구수'`)에서 성별(`'남자'`, `'여자'`)과 연령대(`'0세부터9세'`, ..., `'70세이상'`)를 별도의 독립된 컬럼으로 분리 추출하였습니다.
- **중복 컬럼 제거 (총생활인구수)**: `"총생활인구수"`는 Tidy Data 구조 하에서 기준일/시간대/행정동별 `"생활인구수"`의 합산(`sum()`) 연산으로 언제든 계산할 수 있습니다. 따라서 데이터의 중복성을 완벽히 제거하고 저장 효율을 높이기 위해 최종 데이터프레임 구조에서 **"총생활인구수" 컬럼을 제외**하였습니다.

### 1.2 데이터 타입 최적화 (Downcasting)
행 수가 대폭 늘어난 Tidy Data의 메모리 효율성을 위해 다음과 같이 데이터 타입을 최적화하였습니다.
- `기준일ID` / `행정동코드`: 64비트 정수형(`int64`) $\rightarrow$ 범주형(`category`) (사용자 피드백 반영 최적화)
- `시간대구분`: 64비트 정수형(`int64`) $\rightarrow$ 8비트 정수형(`int8`) (0~23 범위 보존)
- `생활인구수`: 64비트 실수형(`float64`) $\rightarrow$ 32비트 실수형(`float32`) (소수 정밀도 보존 및 다운캐스트)
- `성별` / `연령대`: 문자열형(`object/string`) $\rightarrow$ 범주형(`category`)

## 4. 요약 및 제언
1. **중복성 Zero화 성공**: `"총생활인구수"` 컬럼을 과감히 제거함으로써 데이터의 원자성을 보장(Tidy data 규칙 준수)하는 동시에 용량 절감을 대폭 이루어냈습니다.
2. **카테고리 기법의 극대화**: 행정동코드, 기준일ID를 카테고리화함으로써 Parquet의 컬럼 압축성과 메모리 딕셔너리 인코딩 시너지가 결합되어 극적인 효율 향상을 보였습니다.
3. **분석 제언**: 후속 분석 단계에서 총생활인구수가 필요할 때는 다음과 같이 간결하게 역연산하여 활용할 수 있습니다.
   ```python
   df_total = df_tidy.groupby(['기준일ID', '시간대구분', '행정동코드'])['생활인구수'].sum().reset_name('총생활인구수')
   ```
