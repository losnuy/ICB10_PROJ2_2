우리가 정리한 직무별 핵심 키워드 리스트를 바탕으로, 네이버 API를 호출하여 구직자 관심도를 주간(Weekly) 단위로 정밀하게 추적하고 통합하는 파이썬 자동화 스크립트(fetch_naver_weekly_insights.py)를 작성하고 실행해 줘.

동적 기간 및 주간 주기 설정 (가장 중요):

파이썬 datetime 라이브러리를 활용하여, 스크립트가 실행되는 당일 기준 '전날(어제)'이 포함된 주까지를 조회 종료일로 동적 계산해 줘.

조회 시작일은 2026-01-01부터 설정하되, 데이터의 희소성을 해소하고 거시적 트렌드를 포착할 수 있도록 데이터 수집 주기(time_unit)는 반드시 '주간(week)' 단위로 지정해 줘.

지표 ① (주간 카페 유입량 - 키워드 단독 검색):

특정 키워드가 독취사 등 주요 취업 카페 내에서 언급된 게시글 수를 네이버 카페글 검색 API(cafearticle.json)를 통해 수집해 줘.

카페 데이터는 모수가 적고 희소성이 높으므로, 검색 쿼리에 조합어를 붙이지 않고 오직 [ 키워드 ] (ex. ADsP, 공인노무사) 단독 형태로만 전달하여 누락 없이 최대한 많은 게시글 수를 타겟팅해 줘.

각 게시글의 postdate 필드를 파싱하여, 데이터랩 주기와 일치하도록 주 단위(Weekly) 누적 언급량으로 집계해 줘.

지표 ② (주간 검색 트렌드 추이 - 구직 행동 조합어 검색):

일반적인 자격증명 검색 시 발생하는 전국민적 노이즈(예: 뉴스 검색 등)를 차단하기 위해, 네이버 데이터랩 검색어 트렌드 API에 전달할 주제어(group) 목록은 [자격증/키워드 명칭 + "채용"], [자격증/키워드 명칭 + "스펙"] 등 구직 목적의 행동어 결합 형태로 구성해 줘.

수집 주기는 반드시 time_unit='week'로 설정하여 주간 상대적 검색량 추이를 수집해 줘.

키워드 소스 자동 로드:

우리가 마크다운 파일에 미리 정리해 둔 직무별 핵심 키워드 목록(약 15~20개)을 파이썬 배열(List)로 자동 파싱하여 수집 루프(for문)의 입력값으로 사용해 줘.

네이버 API 요청 루프 및 안전장치:

키워드 배열을 순회하며 네이버 API에 자동으로 요청을 보내는 구조로 짜되, 연속 호출로 인해 네이버 서버로부터 IP 차단(429 Too Many Requests)이 걸리지 않도록 한 번 호출할 때마다 time.sleep(0.5) 안전장치를 반드시 포함해 줘.

API 호출에 필요한 클라이언트 ID와 Secret 값을 입력할 수 있는 변수 자리를 깔끔하게 매핑해 줘.

하이브리드 주간 시계열 데이터 마트 구축 및 고정 경로 저장:

수집된 지표 ①(키워드 단독 주간 카페 유입량)과 지표 ②(조합어 주간 검색 트렌드)를 '주차(Week/Date)'와 '키워드' 기준으로 매핑하여, [주차 시작일(Date), 직무명, 핵심 키워드, 지표 ①(주간 카페 유입량), 지표 ②(주간 검색 트렌드), 최종 취업관심도 지수] 형태의 행과 열을 가진 주간 시계열 데이터프레임 구조로 정형화해 줘.

이때, 두 지표의 균형을 맞춘 파생 변수인 최종 취업관심도 지수 = (지표 ① 주간 유입량) * (지표 ② 주간 검색 트렌드)를 계산하여 컬럼으로 결합해 줘.

스트림릿 대시보드에서 가변 날짜 없이 일관되게 불러올 수 있도록, data/integrated/naver_weekly_insights.json 이라는 고정 경로에 orient='records' 포맷의 JSON 파일로 에러 없이 저장해 줘.
